# Empfohlener Arbeitsablauf

> Dies ist die empfohlene Sequenz für die Erstnutzung. Erfahrene Benutzer können bei jedem Schritt einsteigen.

1. Beginnen Sie im `@clinician`-Modus und beschreiben Sie den Problemkontext in konkreten Begriffen.
2. Fordern Sie einen Plan, ein Skript oder eine Eskalationskarte für die betreffende Routine an.
3. Verwenden Sie optional `@set`, um Ausgangsbedingungen zu injizieren (z. B.: `@set arousal=high medication=wearing-off`).
4. Wechseln Sie zu `@child` und führen Sie die Interaktion als Simulation durch. Aktivieren Sie `@child:inner`, wenn Sie Echtzeit-Zustandsannotationen wünschen.
5. Verwenden Sie `@debrief`, um den Austausch in eine Antezedenz–Verhalten–Konsequenz-Analyse umzuwandeln. Der Zustand des Kindes wird suspendiert, nicht gelöscht.
6. Verwenden Sie `@replay`, um denselben Moment mit einem anderen Ansatz des Erwachsenen zu wiederholen.

Diese Struktur fördert iteratives Üben. Anstatt abstrakt zu fragen „Was soll ich tun?", kann der Benutzer Formulierung, Timing, Übergangshinweise und Wahlarchitektur gegen ein konsistentes Verhaltensmodell testen.
